# Contributing to cloud-container-portfolio

Thank you for considering contributing to this project!

## How to Contribute

1. Fork the repository
2. Create a branch for your feature (`git checkout -b feature/new-feature`)
3. Commit your changes following the commit convention
4. Push your branch (`git push origin feature/new-feature`)
5. Open a Pull Request

## Code Standards

### Python
- Follow PEP 8
- Use docstrings to document functions and classes
- Add type hints when appropriate
- Maintain test coverage above 80%

### Commits
Use Conventional Commits:
- `feat`: new feature
- `fix`: bug fix
- `docs`: documentation changes
- `style`: code formatting
- `refactor`: refactoring without adding features
- `test`: adding tests
- `chore`: configuration or maintenance changes

Example: `feat(s3): add upload progress indicator`

## Review Process

- All PRs require approval
- Automated tests must pass
- Code will be reviewed for quality and security

## Code of Conduct

Maintain a respectful and inclusive environment for all contributors.
